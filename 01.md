Harika! TypeScript öğrenmeye başlamak çok doğru bir karar. JavaScript'e tip güvenliği eklemek, yazılımınızı çok daha sağlam ve okunabilir hale getirir.

"Type oluşturma" aslında **"Verilerimin şeklini (shape) nasıl tanımlarım?"** sorusunun cevabıdır. Hadi birlikte en temelden başlayarak adım adım ilerleyelim.

---

### 0. Başlangıç: Temel İlkel Tipler (Primitives)
Önce en basitinden başlayalım. TypeScript'te değişkenlere doğrudan tip verebilirsiniz. Bunlar zaten var olan tiplerdir:

```typescript
let isim: string = "Ahmet";
let yas: number = 30;
let aktifMi: boolean = true;
```

---

### 1. Adım: Tip Takma Adı (Type Alias) ile Tanımlama
Bir veri yapısını defalarca kullanacaksanız, ona bir **isim** verirsiniz. Bunun için `type` anahtar kelimesi kullanılır.

**Basit kullanım:**
```typescript
// Artık "ID" tipi, string'dir.
type ID = string;

let kullaniciID: ID = "abc-123";
let urunID: ID = "xyz-456";
```

**Nesne (Object) tanımlama:** İşin asıl can alıcı noktası burasıdır. Bir kullanıcının nasıl görünmesi gerektiğini tanımlayalım:
```typescript
type Kullanici = {
  isim: string;
  yas: number;
  email: string;
};

// Artık bu tipi kullanabiliriz:
let ali: Kullanici = {
  isim: "Ali Veli",
  yas: 25,
  email: "ali@mail.com"
};
```
> Eğer `yas` veya `email` alanlarını unutursanız, TypeScript sizi uyarır!

---

### 2. Adım: Arayüz (Interface) ile Tanımlama
`type`'a çok benzeyen bir diğer yöntem de `interface`'dir. Amaç aynıdır: Bir nesnenin şeklini belirlemek.

```typescript
interface KullaniciInterface {
  isim: string;
  yas: number;
  email: string;
}

let ayse: KullaniciInterface = {
  isim: "Ayşe",
  yas: 28,
  email: "ayse@mail.com"
};
```

**🤔 Peki fark ne?** 
Çok temel seviyede şunu bil yeterli: **`type`** daha esnektir (birleşim, kesişim gibi işlemleri kolay yapar). **`interface`** ise daha çok nesne tanımlamak için tercih edilir ve sonradan genişletilebilir (`extends`). Başlangıçta ikisini de kullanabilirsin, ben `type` ile devam edeyim.

---

### 3. Adım: Birleşim Tipleri (Union Types) ile Esneklik
Bir değişkenin **birden fazla** farklı türde olabileceğini söylemek istersen `|` (pipe) işaretini kullanırsın.

```typescript
// Bu değişken ya string ya da number olabilir:
type ID = string | number;

let id1: ID = "abc123"; // Geçerli
let id2: ID = 456;      // Geçerli
// let id3: ID = true;  // HATA! boolean kabul değil.
```

**En çok işe yarayan hali:** Sabit değerlerle (Literal) kullanmaktır.
```typescript
type SiparisDurumu = "beklemede" | "hazırlanıyor" | "kargoda" | "teslim edildi";

let durum: SiparisDurumu = "kargoda"; // Geçerli
// durum = "iptal"; // HATA! Sadece yukarıdaki 4 değerden biri olabilir.
```

---

### 4. Adım: Kesişim Tipleri (Intersection Types) ile Birleştirme
Eğer iki ayrı tipi birleştirip tek bir tip oluşturmak istersen `&` (ampersand) kullanırsın. 
*Düşünelim: Bir "Kullanıcı" tipim var, bir de "Admin" özelliklerim var. İkisini birleştirelim.*

```typescript
type TemelKullanici = {
  isim: string;
  yas: number;
};

type AdminOzellikleri = {
  yetkiSeviyesi: number;
  sifre: string;
};

// İkisini birleştiren yeni tip:
type Admin = TemelKullanici & AdminOzellikleri;

let superAdmin: Admin = {
  isim: "Zeynep",
  yas: 32,
  yetkiSeviyesi: 5,
  sifre: "123456"
};
```

---

### 5. Adım: Diziler (Arrays) ve Tuple (Sabit Uzunluklu Dizi)
- **Dizi:** Bir tipin dizisi olduğunu belirtmek için `[]` veya `Array<T>` kullanılır.
```typescript
let isimListesi: string[] = ["Ali", "Ayşe", "Mehmet"];
let yasListesi: Array<number> = [25, 30, 28];
```

- **Tuple:** Dizinin uzunluğunu ve *hangi sırada hangi tipin geleceğini* sabitlemek istersen kullanılır.
```typescript
// 0. eleman string, 1. eleman number, 2. eleman boolean OLMALI:
type KisiBilgisi = [string, number, boolean];

let veri: KisiBilgisi = ["Ahmet", 30, true]; // Geçerli
// let veri2: KisiBilgisi = [30, "Ahmet", true]; // HATA! Sıralama yanlış.
```

---

### 6. Adım: Opsiyonel (İsteğe Bağlı) ve Readonly (Salt Okunur) Alanlar
Bir alanın zorunlu olmamasını istersen `?` koyarsın. Değişmemesini istersen `readonly` koyarsın.

```typescript
type Araba = {
  readonly plaka: string; // Bir kere atanır, sonra değişmez!
  marka: string;
  model: string;
  yil?: number; // Bu alan opsiyonel, atamak zorunda değilsin.
};

let benimArabam: Araba = {
  plaka: "34 ABC 123",
  marka: "BMW",
  model: "3.20"
  // yil yok, sorun değil.
};

// benimArabam.plaka = "35 XYZ 456"; // HATA! Çünkü readonly.
```

### 7. Adım: `any` ve `unknown` (Bunlara Dikkat!)
- **`any`**: "Tip denetimini tamamen kapat, ne gelirse gelsin" demektir. **Kullanmaktan kaçının!** TypeScript'in amacını bozar.
- **`unknown`**: "Ne geleceğini bilmiyorum ama güvenli oynamak istiyorum" demektir. Kullanmadan önce tip kontrolü yapmanı zorunlu kılar.

```typescript
let guvenliDegil: any = 5;
guvenliDegil = "Artık string oldu"; // Sıkıntı yok (kötü).

let bilinmiyor: unknown = "Merhaba";
// bilinmiyor.toUpperCase(); // HATA! unknown olduğu için metot çağıramazsın.
if (typeof bilinmiyor === "string") {
  console.log(bilinmiyor.toUpperCase()); // Burada çalışır.
}
```

### ⭐ Hepsini Birleştiren Mini Örnek
Şimdi öğrendiklerimizi tek bir yerde kullanalım:

```typescript
// 1. Literal ve Union birleşimi
type Cinsiyet = "Erkek" | "Kadın";

// 2. Type oluştur
type Musteri = {
  readonly id: number;
  ad: string;
  cinsiyet: Cinsiyet; // Sadece "Erkek" veya "Kadın"
  yas?: number; // Opsiyonel
};

// 3. Type oluştur (Dizi ve Tuple içeren)
type Siparis = {
  musteri: Musteri;
  urunler: string[]; // Dizi
  odemeBilgisi: [string, number]; // Tuple (kart_tipi, son_4_hane)
};

// Kullanımı
const yeniSiparis: Siparis = {
  musteri: {
    id: 1,
    ad: "Mehmet",
    cinsiyet: "Erkek",
  },
  urunler: ["Laptop", "Fare"],
  odemeBilgisi: ["Visa", 1234],
};

console.log(yeniSiparis);
```
